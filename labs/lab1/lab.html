Introduction

In this lab, a PCB with FPGA and MCU was assembled andused to drive a 7 segment LED display. Several onboard components were tested, ensuring proper communication between both microcontrollers.

Design and Testing Methodology

The tests for this project exist under two realms - simulation and hardware.

Hardware tests

Three hardware tests were conducted. The first ensured connection between the FPGA and the on-board LEDs, the second required the FPGA to pull a pin on the MCU high, each driving ledâ€™s depending on their current state. The third test was using pins on the PCB to control a number displayed on the 7 segment display, as well as on-board LEDS to understand the internal state.

Simulation goals

The 7 segment display code was verified first in simulation due to the extra complexity required. For the simulation, a test bench for each possible user input (switches representing binary numbers 0-9) was constructed.

Technical Documentation

Block Diagram

The block diagram for the 7 segment display is shown below. This describes the internal structure of the FPGA.

The FPGA controlls the main logic. It takes as input a binary number from 0-9 and converts it to control for two the on board leds, and drives a third by a 2.4 hz clock signal.

The FPGA also converts the input binary number to contorl over the 7 segment display to show the input number in decimal form.

Results and Discussion

This breif series of tests (15 hours? More?) showed that the hardware is working as expected; The FPGA and MCU were able to properly communicate, and a 7 segment display was properly driven. Further, a 2.4 hz clock signal was generated and used to drive an onboard LED.

All intended tasks were completed successfully. The design could be improved to provide adaptive current. A 7 segment display driven by connections to a constant voltage would allow better control of LED brightness. The present setup distributes current across all leds in the 7 segment display, which decreases the brightness.
